Codeforces 1106B Lunar New Year and Food Ordering

题目大意: 有 nnn 个物品，第 iii 个单价为 cic_ici​ ，持有 aia_iai​ 件。有 mmm 次破坏性询问(即会改变现在物品持有数量): 有一个人要买 djd_jdj​ 个物品 tjt_jtj​ 。如果够了就直接买，不够就从其他物品中选择最便宜且编号尽可能小的物品买直到买到 djd_jdj​ 个，如果所有物品总量都不够 djd_jdj​ ，这个人就逃单了(拿走所有物品不付钱)。求每个人的花费。 n,m≤105,ci≤106n,m \le 10^5, c_i \le 10^6n,m≤105,ci​≤106 。


显然，用来补订单的物品是有顺序的: 价格递增的前提下编号递增。所以只需要将所有物品的编号按照 (ci,i)(c_i,i)(ci​,i) 为关键字排序，然后用一个单调指针扫描现在该用什么补订单就可以了。

虽然每个人都要补订单的话复杂度乍一看是 O(nm)O(nm)O(nm) ，但是因为每一次补订单相当于一种物品卖光了，均摊补订单次数只有 O(m)O(m)O(m) (可以理解为指针单调递增)。总复杂度 O(nm)O(nm)O(nm) 。


Codeforces 1106C Lunar New Year and Number Division

题目大意: 将 nnn 个数分为若干组，每组不少于 222 个数，最小化【每组数的和的平方】的和。 2≤n≤3×105,n≡0(mod2)2 \le n \le 3 \times 10^5, n \equiv 0 \pmod 22≤n≤3×105,n≡0(mod2) 。


显然是两两一组。考虑怎么分最优秀。乱搞选手当然可以一眼就看出怎么分最优秀。

设分组方案为 $$(a_1,b_1), (a_2,b_2), …, (a_{n/2},b_{n/2})$$ ，那么答案是:

∑i=1n2(ai+bi)2=∑i=1n2(ai2+2aibi+bi2)\sum_{i=1}^\frac{n}{2} (a_i+b_i)^2 = \sum_{i=1}^\frac{n}{2} (a_i^2 + 2 a_i b_i +b_i^2) i=1∑2n​​(ai​+bi​)2=i=1∑2n​​(ai2​+2ai​bi​+bi2​)

因为 ∑ai2\sum a_i^2∑ai2​ 和 ∑bi2\sum b_i^2∑bi2​ 都是定值，只需要最小化 ∑aibi\sum a_ib_i∑ai​bi​ 。这就是排序不等式: 排序不等式。即最大的 aia_iai​ 和最小的 bib_ibi​ 配对，次大的 aia_iai​ 和次小的 bib_ibi​ 配对，…，以此类推，答案最小。

那么怎么证明 aia_iai​ 和 bib_ibi​ 要交叉着分？

去翻了一下题解，题解这部分的证明是没有的，感觉不太严谨。

可以考虑将序列 aia_iai​ 和 bib_ibi​ 都看做 nnn 个数的完整集合，这时候最优配对方案就是逆序和。而任意一个将 aia_iai​ 和 $b_i $ 分组配对的方案都可以对应一个完整集合方案(但是反之不一定)。例如1 2 3 4 5 6的一个分组配对方案是3-1 2-4 5-6，可以对应构造出完整集合方案:

而任意一个完整集合方案都不如逆序和更优。而逆序和恰好可以构造出一个分组配对方案。因此所有分组配对方案中交叉配对逆序和最优。即: 即一个分组配对方案为 AiA_iAi​ ，交叉配对方案为 A∗A^*A∗ ，一个分组配对方案对应的完整集合方案为 Bi=f(Ai)B_i = f(A_i)Bi​=f(Ai​) 。

cost(Ai)=12cost(f(Ai))=12cost(Bi)≥12cost(B∗)=cost(f−1(B∗))=cost(A∗)cost(A_i) = \frac{1}{2}cost(f(A_i)) = \frac{1}{2}cost(B_i) \ge \\ \frac{1}{2} cost(B^*) = cost(f^{-1}(B^*)) = cost(A^*) cost(Ai​)=21​cost(f(Ai​))=21​cost(Bi​)≥21​cost(B∗)=cost(f−1(B∗))=cost(A∗)

其中，最后一个等式有意义当且仅当 $$f{-1}(B) = A^$$ 存在，而其的确存在。因此证毕。

复杂度 O(nlog⁡n)O(n \log n)O(nlogn) ，瓶颈在于排序(当然可以桶排 O(n+A)O(n+A)O(n+A) )。


Codeforces 1106D Lunar New Year and a Wander

题目大意: 给定一张 nnn 点 mmm 边连通无向图，走一条可经过重复点和边的路径，每到达一个没到达过的节点时写下这个节点，求字典序最小的路径。 n,m≤105n,m \le 10^5n,m≤105 。


怎么感觉是个经典老题？

从 111 号点开始走，访问过的点一定是连通块，因为可以走重复点和边，因此每次可以走到连通块边界上的任何一个点。那么只需要维护当前连通块外围相邻的所有点，用一个优先队列排序每次贪心将最小的点加入连通块再扩展外围就行了。

复杂度 O(mlog⁡n)O(m \log n)O(mlogn) 。


Codeforces 1106E Lunar New Year and Red Envelopes

DPDPDP 可以一眼瞅出来: f[i][j]f[i][j]f[i][j] 表示当前在点 iii ，还剩 jjj 次强制机会，最小收益是多少，在点 iii 可以选择强制或不强制:

f[i][j]=min⁡(f[i+1][j−1],f[go(i)][j]+w(i))f[i][j] = \min(f[i+1][j-1],f[go(i)][j]+w(i)) f[i][j]=min(f[i+1][j−1],f[go(i)][j]+w(i))

复杂度 O(nlog⁡n+nm)O(n \log n +nm)O(nlogn+nm) 。
