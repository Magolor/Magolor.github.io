Codeforces 1191D Tokitsukaze, CSL and Stone Game

题目大意: 有nnn堆石子，两个人轮流一次取一个石子，任何一个玩家取走石子后不能有两堆石子数目相同。给定一个局面求获胜方。注意000个石子的石子堆也是石子堆，不能数目相同，注意初始条件下可以有两堆石子数目相同，只要第一次移动后不同即可。n≤105,0≤ai≤109n \le 10^5, 0 \le a_i \le 10^9n≤105,0≤ai​≤109


分类讨论下:

  • 初始局面合法

    结束局面一定是0,1,2,3,...,n−10,1,2,3,...,n-10,1,2,3,...,n−1: 反证法，如果有别的结束局面，排序后第iii项不是iii，那么一定可以操作。因此只需要统计初始局面总石子数减去结束局面总石子数的奇偶性就行了。

  • 初始局面不合法

    • 有三个相同数目或者多个两对数目相同

        • 两堆000。

          先手也凉了，输出cslnb。

        • 形如x−1,x,xx-1,x,xx−1,x,x。

          先手还是凉了，输出cslnb。

      • 第一步操作可以可以消除掉这两堆

Codeforces 1191E Tokitsukaze and Duel

题目大意: 长度为nnn的01序列。两人轮流每次给一段长度恰好为kkk的区间染色，染成同色者胜。求获胜方或平局(无限循环)。1≤k≤n≤1051 \le k \le n \le 10^51≤k≤n≤105。


看上去是个挺复杂的问题，手动操作一下试一试。(假设先手玩家为A，后手玩家为B)第一步正常，第二步正常，问题出现在第三步。

假设在第三步，A经过操作获胜了。那么第二步，B就是必输状态，但是B完全可以重复A在第一步的操作从而把相同的状态转移给第三步的A，第三步的A就是必输状态，这个假设矛盾。

然后同理B也不可能在第四步胜利，因为A可以将第三步的状态原封不动传给第四步的B。以此类推，只要第二步游戏没结束，就结束不了了。

第一步A能不能一步胜利很好判断: 就是枚举长为kkk的区间，如果存在一个区间外的前缀和后缀都是相同的，那么就可以胜利。这个过程是O(n)O(n)O(n)的。

显然，如果第一步A不能胜利。枚举第一步A的操作，判断第二步B能不能一步胜利，可以得到一个O(n2)O(n^2)O(n2)的算法。考虑优化。

判断一步胜利的方法，如果已经预处理出最长前缀和最长后缀，只需要判断两者长度和+kkk是不是大于序列总长度即可。那么枚举第一步A的选择后，如果已经对于每个位置求出这个位置向右延伸的最长距离，就可以O(1)O(1)O(1)更新最长前缀和最长后缀。就做完了。

复杂度O(n)O(n)O(n)。Python的边界处理真的难受，所以还是搞成数组下标从1开始了。

Codeforces 1191F Tokitsukaze and Strange Rectangle

题目大意: 给定平面上nnn个点，每次可以用x∈[l,r],y∈[a,+∞)x \in [l,r],y \in [a,+\infty)x∈[l,r],y∈[a,+∞)作为限制条件来得到一个点集。求能得到多少种不同的点集。n≤105,x,y≤109n \le 10^5, x,y \le 10^9n≤105,x,y≤109


冷静一下: 如果xxx和yyy互不相同，那么任何一个点集由下端点、左端点和右端点唯一确定，且左右端点高于下端点。那么就有一个很简单的做法，对于每个点统计其左上方点数lll和右上方点数rrr，以这个点为下顶点的点集就有(l+1)∗(r+1)(l+1)*(r+1)(l+1)∗(r+1)个。统计一个点一侧的点数就是个二维数点，扫描线+离散化+树状数组就行了。O(nlog⁡n)O(n \log n)O(nlogn)。

现在有相同的xxx和yyy。发现相同的yyy好处理，因为根本不需要处理，相同的yyy对答案没啥影响。相同的xxx就会出现重复问题: 例如，假设某个xxx值有若干个点，枚举第iii个点作为下端点的时候，其左端点就不能在第i−1i-1i−1个点左侧，因为那样第i−1i-1i−1个点也就成为下端点了。

具体地，枚举第i−1i-1i−1个点作为下端点计算完，枚举第iii个点的时候要统计所有以iii为下端点且不包含i−1i-1i−1的点，就需要l[i] -= l[i-1]。

时间复杂度O(nlog⁡n)O(n \log n)O(nlogn)。担心Python过不去先写了个C++版本。

C++版本140ms140\text{ms}140ms，时限是3000ms3000\text{ms}3000ms，即使Python慢20倍也够了。于是大胆进行Python实现。
